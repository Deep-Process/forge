# /discover $ARGUMENTS

Discovery phase — explore options, assess risks, and prepare architecture BEFORE planning.

## Arguments

| Form | Meaning | Example |
|------|---------|---------|
| `{topic}` | Explore a topic / direction | `/discover authentication for the API` |
| `{topic} --full` | Run all 4 analyses | `/discover migration to microservices --full` |
| `{topic} --risk-only` | Run only risk analysis | `/discover switching to Redis --risk-only` |

## Instructions

Read and follow the procedure in `skills/discover/SKILL.md`.

Key points:
- Deep-* skills provide methodology; discover coordinates and records findings
- Reference parent alignment (idea/objective) — don't re-align
- Record findings as OPEN decisions in Forge
- If feasibility = NO-GO, STOP and present to user
- Present a Discovery Brief at the end with clear next steps
